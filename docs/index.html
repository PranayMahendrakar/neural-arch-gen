Generated Architecture Families

CNN

AutoCNN

For small-to-medium image classification tasks (<50k samples).

▸Conv2d(in->32, 3x3) + BN + ReLU + MaxPool
▸Conv2d(32->64, 3x3) + BN + ReLU + MaxPool
▸Conv2d(64->128, 3x3) + BN + ReLU + MaxPool
▸AdaptiveAvgPool(4x4) + Flatten
▸Linear(2048->256) + Dropout(0.5)
▸Linear(256->num_classes)
ResNet

AutoResNet

For larger classification datasets. Residual skip connections prevent vanishing gradients.

▸Stem: Conv7x7 + BN + ReLU + MaxPool
▸Stage 1: N× ResBlock(64)
▸Stage 2: N× ResBlock(128)
▸Stage 3: N× ResBlock(256)
▸GlobalAvgPool + FC(256->num_classes)
U-Net

AutoUNet

Encoder-decoder with skip connections for semantic segmentation.

▸Encoder: DoubleConv blocks with MaxPool downsampling
▸Bottleneck: DoubleConv(512->1024)
▸Decoder: ConvTranspose2d upsampling + skip concat
▸Head: Conv1x1 -> num_classes channels
Transformer

AutoTransformer

Token-based self-attention for NLP classification.

▸TokenEmbedding(vocab->d_model) + PositionalEmbed
▸TransformerEncoder × N layers (multi-head attn + FFN)
▸Mean-pool over sequence
▸FC(d_model->num_classes)
LSTM

AutoLSTM

Recurrent architecture for time-series and sequential data.

▸LSTM(input_size, hidden=64/128, num_layers=1/2)
▸Dropout(0.2) between layers
▸Take last hidden state
▸FC(hidden->output)
MLP

AutoMLP

Fully-connected network for tabular data and regression.

▸Flatten(input_shape)
▸Linear + BN + ReLU + Dropout (× depth)
▸Hidden sizes scale with dataset tier
▸Linear(hidden->output)

Training Pipeline

🔄 Training Loop Features

  • Mixed-precision (AMP) with GradScaler for GPU
  • Gradient clipping (max norm=1.0)
  • Linear warmup + CosineAnnealingLR scheduler
  • SequentialLR combining warmup and cosine
  • Early stopping with configurable patience
  • Best-checkpoint saving by validation loss
  • Per-epoch logging with loss and accuracy

🧪 Regularisation Strategies

  • dropout 0.3 for small datasets
  • weight_decay 1e-4 to 1e-2
  • early_stopping for tiny/small
  • label_smoothing for large datasets
  • data_augmentation for image tasks
  • BatchNorm on all conv/linear layers

Architecture Search Methodology

🧠 Rule-Based Neural Architecture Search (RBNAS)

Unlike gradient-based NAS (DARTS) or evolutionary NAS, this system uses a deterministic rule engine built on domain expertise. The philosophy is that most practical ML problems can be solved well with a small set of well-tested architecture families — the key insight is selecting the right family and scaling it correctly.

The search process runs in under one second and requires no GPU, making it ideal for rapid prototyping. It consists of four sequential decisions: family selection, depth/width scaling, regularisation selection, and hyperparameter derivation.

Step 1: Problem Classification

The ProblemAnalyzer maps (problem_type, dataset_size) to a complexity tier (tiny/small/medium/large) using fixed thresholds at 1k, 10k, and 100k samples. This tier drives all downstream decisions.

Step 2: Family Selection

Architecture family is determined by problem type first, then refined by dataset size. Classification → CNN (small) or ResNet (large). Segmentation → U-Net. NLP → Transformer. Time-series → LSTM or TCN. Regression → MLP.

Step 3: Depth & Width Scaling

Within each family, filter counts, hidden sizes, and number of layers are scaled by the complexity tier. Tiny datasets get shallow, narrow networks; large datasets get deep, wide ones. This prevents both overfitting (small data + big model) and underfitting (large data + small model).

Step 4: Training Config Derivation

Learning rate, batch size, regularisation strength, and scheduler type are derived from the architecture family and complexity tier using lookup tables grounded in published best practices. Mixed precision and gradient clipping are enabled automatically for GPU hardware.
